PROCEDURE
Procedures are used to perform an action 

FUNCTION
Functions are used to perform a calculation and return a value

PARAMETER
Complex sub-programs have parameters to pass values or information into the sub-program. They enable to sub-program to be flexible

There are three parts of the parameter, 
1. parameter_name --> the name of the parameter
2. parameter_mode --> IN = read only
3. datatype --> the data type of the parameter

EXAMPLE OF A PROCEDURE
Suppose we want to display "Hello Universe!" Instead of "Hello World!", we can keep the string "Hello" static and parametric the "world or universe" part.

CREATE OR REPLACE PROCEDURE HELLO_PLACE (IN_PLACE in VARCHAR2)
IS
	L_MESSAGE VARCHAR2(100);
BEGIN
	L_MESSAGE := 'HELLO '|| IN_PLACE;
	DBMS_OUTPUT.put_line(L_MESSAGE);
End HELLO_PLACE;

BEGIN
	HELLO_PLACE('WORLD!!');
	HELLO_PLACE('UNIVERSE!!');
END;

EXAMPLE OF A FUNCTION
CREATE OR REPLACE FUNCTION HELLO_PLACE (IN_PLACE IN VARCHAR2)
	RETURN VARCHAR2;
IS
BEGIN
	RETURN 'HELLO '||IN_PLACE;	
END;

DIFFERENCE BETWEEN PROCEDURE AND FUNCTION BASED ON THE ABOVE EXAMPLES:
1. The function defines the value being returned and not the action being performed.
2. The body of the function has a RETURN statement which returns the operation being performed.
3. The RETURN statement in the creation block of the function, specifies the data type of value being returned.
